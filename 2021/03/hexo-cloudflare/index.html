使用Cloudflare对你的网站进行全局CDN加速(未写完)

发表于2021-03-22|更新于2021-03-23|Hexo
|字数总计:451|阅读时长:1分钟|阅读量:

前言

Cloudflare免费提供cdn服务即可以使你的网站访问加速，并且自定义域名提供免费的 https 服务而不会通过域名访问时显示网站不安全的提示。

使用Cloudflare的前先搭建好你的博客并且部署到你的Github仓库上，可参考Github+hexo搭建个人博客

使用Cloudflare的前提是要有一个域名，没有域名可以看这篇文章：Hexo博客使用vercel进行全局加速

Cloudflare不建议和vercel一起用！

注册Cloudflare

  • 进入Cloudflare官网注册一个账号，页面是中文的就不多介绍了。

绑定域名

绑定域名之前建议先到域名控制台先把所有的解析删掉

  1. 注册号账号后回到主页上，点击添加站点，里面输入你购买的域名地址
  2. 选择计划选择免费的就好
  3. 在查看DNS界面直接点下一步就好
  4. 修改域名的DNS服务器，改成Cloudflare提供的，怎么改就不介绍了，每个注册机构更改方式都不同。


  5. 修改好后点击检查服务器，一般刷新时间不会太长，小到几秒钟，最高应该不会超过五分钟。
  6. 接下来在快速入门指南里点摘要，建议里面的都选上
  7. 设置好后点完成，这个是时候域名就绑定完成了

在Cloudflare添加解析

类型名称目标
CNAME
@或者你想要的子域名
Github Page的链接

添加好记录后等待5-10分钟基本域名就可以访问了。
CloudFlare的缓存刷新是比较慢的，最高要48小时，如果要调试功能需要实时刷新可以打开“开发模式”

至此就搭建完毕了
